- 原因：
  - 用户要求重新读取工作区 Markdown 后，在 NEMU 已能跑 RT-Thread 的基础上，实现 NPC 五级流水线。
  - 本阶段要求：不带 icache/dcache、不带分支预测、不带 forwarding；用 Verilator 编译 RTL 并在类似 NEMU 的仿真环境运行；接 NEMU difftest，每提交一条指令比较 PC、32 个 GPR 和 CSR；中间目标先跑通 `am-kernels/tests/cpu-tests`。
- 修改内容摘要：
  - 新增 `NpcCore.v`，实现 IF/ID/EX/MEM/WB 五级流水线 RV32IM/Zicsr 子集。
  - RAW hazard 在 ID 阶段停顿，直到 ID/EX、EX/MEM、MEM/WB 中相关写回离开；没有前递。
  - 分支、`jal/jalr`、`ecall/mret` 在 EX 阶段决定跳转，flush 年轻级；没有分支预测。
  - 使用组合仿真取指/访存端口，不实现 icache/dcache 和 AXI，本阶段先服务 Verilator + cpu-tests 闭环。

  - C++ 仿真通过 `dlopen()` 加载 NEMU reference `.so`，每个 WB commit 调用 `difftest_exec(1)`，比较 next PC、32 个 GPR、`mstatus/mtvec/mepc/mcause/mie/mip/mscratch`。
  - 补全 NEMU reference 导出接口：`difftest_memcpy/regcpy/exec/raise_intr/init`。
  - `difftest_memcpy()` 对 0 长度拷贝直接返回，避免 `addr + n - 1` 下溢。
  - 扩展 RISC-V difftest register size 到 GPR + PC + machine CSR，并补齐 NEMU 作为 DUT 时的 RISC-V checkregs。
  - 新增 `riscv32-npc` AM 构建入口，使用 `-march=rv32im_zicsr -mabi=ilp32`。
